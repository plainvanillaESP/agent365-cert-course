# Módulo 12 — Monitorización, auditoría y reporting con Microsoft Defender XDR

**Duración estimada:** 90 min lectura + ~30 min quiz y labs (cuando se produzcan).
**Área del examen:** 5 (Monitor, troubleshoot and improve).
**Estado:** En producción. Teoría completada en Fase E.5; quiz de práctica y laboratorios pendientes en Fase E.6.

## Archivos del módulo

- [`teoria.md`](./teoria.md) — Contenido teórico completo (7 secciones, 90 min lectura).
- [`recursos.md`](./recursos.md) — URLs de Microsoft Learn, repositorios de queries, blogs oficiales y lecturas adicionales.
- `quiz-practica.md` — *(pendiente Fase E.6)* — preguntas Q-12-N de práctica.
- `laboratorios.md` — *(pendiente Fase E.6)* — 4 labs prácticos.
- `module.yaml` — manifiesto del módulo con OAs, glosario y secciones.

## Objetivos de aprendizaje

Al finalizar el módulo el alumno será capaz de:

1. Posicionar Defender XDR como capa detectiva en el ecosistema Agent 365.
2. Navegar las 5 superficies operativas del portal Defender XDR.
3. Consultar `CloudAppEvents` con KQL para investigar agentes.
4. Construir queries para 3 patrones problemáticos canónicos.
5. Configurar custom detection rules con severidad calibrada.
6. Integrar Defender XDR con Microsoft Sentinel.
7. Investigar un incidente real con timeline + correlación + contención.

## Prerrequisitos

- Módulo 01 (fundamentos de Agent 365).
- Módulo 06 (Microsoft Entra Agent ID).
- Módulo 09 (Permisos y Conditional Access).
- Módulo 10 (Purview Information Protection).
- Módulo 11 (DLP y Communication Compliance).

Familiaridad básica con KQL es prerrequisito de facto. Si no la tienes, dedicar 2-3 horas al tutorial oficial de Microsoft Learn es la mejor inversión antes de empezar.
