# Módulo 09 — Permisos, accesos y Conditional Access para agentes

**Duración estimada:** 75 min de lectura + ~30 min de quiz y labs (cuando se produzcan).
**Área del examen:** 2 (Manage agent identity, sponsorship and lifecycle).
**Estado:** En producción. Teoría completada en Fase B.1; quiz de práctica y laboratorios pendientes en Fase B.2.

## Archivos del módulo

- [`teoria.md`](./teoria.md) — Contenido teórico completo (6 secciones, 75 min lectura).
- [`recursos.md`](./recursos.md) — URLs de Microsoft Learn, blogs oficiales y lecturas adicionales.
- `quiz-practica.md` — *(pendiente Fase B.2)* — 8 preguntas Q-09-N de práctica.
- `laboratorios.md` — *(pendiente Fase B.2)* — 4 labs prácticos.
- `module.yaml` — manifiesto del módulo con OAs, glosario y secciones.

## Objetivos de aprendizaje

Al finalizar el módulo el alumno será capaz de:

1. Distinguir los cuatro tipos de permisos que un agente puede tener (blueprint, OBO, application, custom security attributes).
2. Configurar Conditional Access policies sobre workload identities.
3. Aplicar políticas basadas en risk score con Identity Protection.
4. Reconocer las seis detecciones específicas para agentes.
5. Diferenciar trazabilidad OBO vs own identity en sign-in logs.
6. Diseñar políticas de acceso end-to-end combinando blueprint + CA + ID Protection.
